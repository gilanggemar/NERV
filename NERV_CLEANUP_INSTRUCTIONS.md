## PHASE 6 — Optional Dependency Optimization (REDUCES npm install TIME)

These are NOT required for the GitHub push, but will reduce your `node_modules` size by ~100 MB on every developer machine. Execute these ONLY if you want a leaner project.

### 6A — Consolidate duplicate icon libraries

You currently have BOTH `@tabler/icons-react` (55 MB) and `lucide-react` (35 MB). You need to pick ONE and migrate all imports from the other.

**Recommended approach:** Keep `lucide-react` (it's lighter, more common in shadcn/ui projects, and already used by your 28 shadcn/ui components).

**Migration steps:**

1. Search every file in `dashboard/components/` and `dashboard/app/` for imports from `@tabler/icons-react`.
2. For each Tabler icon import, find the equivalent icon name in `lucide-react`. Most icon names are identical or very similar (e.g., `IconBrain` in Tabler → `Brain` in Lucide).
3. Replace the import statement. Example:
   - **Before:** `import { IconBrain } from '@tabler/icons-react'`  
   - **After:** `import { Brain } from 'lucide-react'`
4. Update the JSX usage. Tabler uses `<IconBrain size={20} />` while Lucide uses `<Brain size={20} />`.
5. After ALL Tabler imports are replaced, uninstall the package:
   ```powershell
   cd "D:\AI Model\2-Antigravity Projects\NERV.OS\dashboard"
   npm uninstall @tabler/icons-react
   ```

**CRITICAL:** Do NOT uninstall `@tabler/icons-react` until EVERY import has been migrated. Search thoroughly. A single missed import will cause a build error.

### 6B — Consolidate duplicate animation libraries

You have both `framer-motion` and `motion` installed. `motion` is the renamed successor of `framer-motion` (same library, same API). Keep only `motion`.

1. Search all files for `import ... from 'framer-motion'`.
2. Replace with `import ... from 'motion/react'` (the new import path).
3. After ALL imports are migrated:
   ```powershell
   npm uninstall framer-motion
   ```

**NOTE:** The API is identical. Only the import path changes.
